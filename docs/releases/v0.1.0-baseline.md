## Architecture Findings

### ✅ Solid

1. **Generator registry** — single canonical registry, barrel exports, data-driven
2. **Canonical render path** — preview/export/batch all use `renderToTarget`
3. **Recipe validation** — zod schema, atomic parse→validate→apply
4. **Resolution system** — data-driven presets, custom dimensions, aspect-ratio lock
5. **Import safety** — JSON parsing isolated, schema validation before mutation
6. **PRNG** — seeded, deterministic, no `Math.random()` in generators

### ⚠️ Issues Found

1. **Escape key resets state instead of closing mobile sheet** — `KeyboardShortcuts.tsx:18-20` calls `s.hydrate({})` which merges empty object (no-op), doesn't close BottomSheet
2. **No Cmd+S / Ctrl+S shortcut** — not implemented
3. **No Cmd+Shift+S / Ctrl+Shift+S shortcut** — not implemented
4. **No drag-and-drop recipe import** — not implemented
5. **No error boundary** — React errors will show raw stack traces
6. **No WebGL fallback** — fluid-gradient will crash silently if WebGL unavailable
7. **No export-size limits** — user can request 15360×15360 and crash the browser
8. **No duplicate-export prevention** — repeated clicks create overlapping downloads
9. **`persistToLocalStorage` runs every render** — no debouncing, writes on every state change
10. **Unused imports** — `getDefaultParams` in page.tsx, `CURATED_PALETTES` in PalettePicker.tsx
11. **`resolvePalette` is a no-op** — always returns raw palette regardless of mode
12. **No `vercel.json`** — may need security headers, cache config
13. **No README** — no project documentation
14. **No CHANGELOG** — no release notes
15. **`math.random()` used in `randomizeSeed()`** — acceptable (seed generation, not rendering), but worth noting
