## 1. Application Deployment Architecture & Route Inspection

WallMyDevice is built on Next.js 14.2.35. Inspection confirms:
- **App Router Routes**: Single static home route (`/` in `app/page.tsx`).
- **Server Functions & Actions**: None.
- **Image Optimizer API (`next/image`)**: Unused. All wallpapers render directly to HTML5 `<canvas>` / WebGL context.
- **Middleware & Rewrites**: None (`next.config.mjs` contains no proxy rewrites or middleware).
- **WebSockets / Server Push**: None.

---

## 2. Evidence Categorization

### A. Support-Policy Evidence
- Next.js 14 has reached End of Life (EOL) for security updates. Next.js major release lines 15+ and 16+ receive active security maintenance.

### B. `npm audit` Evidence
Running `npm audit` reports 5 total vulnerabilities across `next`, `postcss`, and `eslint-config-next`:
- 1 Moderate Severity
- 4 High Severity
- 0 Critical Severity

### C. Published Security-Advisory Evidence
- `minimatch` (ReDoS in Glob Pattern Matching): Remediated via `npm audit fix` (installed version 9.0.7).
- `glob` (CLI Command Injection): Transitive dev dependency (`eslint-config-next`). Unused CLI flags in build scripts.
- `next` (RSC, Proxy, WebSocket, CSP advisories): Unused server-side features in client-rendered canvas static export.
- `postcss` (Style Tag XSS): Transitive build-time sub-dependency of `next`.
