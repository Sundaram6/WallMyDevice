## Deployment Architecture & Route Inspection

WallMyDevice is built on Next.js 14.2.35. A structural inspection of the codebase confirms:
- **App Router Routes**: Single static home route (`/` in `app/page.tsx`).
- **Server Functions & Actions**: None.
- **Image Optimizer API (`next/image`)**: Unused. All wallpapers render to HTML5 `<canvas>` / WebGL context.
- **Middleware & Rewrites**: None (`next.config.mjs` contains no proxy rewrites or middleware).
- **WebSockets / Server Push**: None.

---

## Security Advisory Inventory & Patch Analysis

### Official Next.js July Security Release Notice
The official Next.js security release addresses multiple high- and medium-severity advisories across RSC deserialization, Image Optimizer, proxy bypass, and SSRF.
Patched supported framework releases are:
- `next@16.2.11` (Active LTS)
- `next@15.5.21` (Maintenance LTS)

Next.js 14 is classified as unsupported and did not receive backported July security patches.

## Detailed Advisory Inventory

### 1. `minimatch` — ReDoS in Glob Pattern Matching
- **Advisory IDs**: GHSA-3ppc-4f35-3m26, GHSA-7r86-cg39-jmmj, GHSA-23c5-xmqv-rm74
- **Package**: `minimatch` (9.0.0 - 9.0.6)
- **Installed Version**: 9.0.7 (Remediated)
- **Dependency Path**: `@typescript-eslint/parser` → `@typescript-eslint/typescript-estree` → `minimatch`
- **Severity**: High
- **Affected Feature**: Regular expression backtracking in pattern matching.
- **Patched Version**: 9.0.7+
- **Usage in WallMyDevice**: Dev dependency for ESLint and TypeScript parsing.
- **Vercel Applicability**: None (build-time tooling).
- **Final Disposition**: **REMEDIATED** via `npm audit fix`.

### 2. `glob` — CLI Command Injection
- **Advisory ID**: GHSA-5j98-mcp5-4vw2
- **Package**: `glob` (10.2.0 - 10.4.5)
- **Installed Version**: 10.3.10
- **Dependency Path**: `eslint-config-next` → `@next/eslint-plugin-next` → `glob`
- **Severity**: High
- **Affected Feature**: Command injection via `-c` / `--cmd` flags in glob CLI.
- **Patched Version**: 10.4.6+
- **Usage in WallMyDevice**: Dev dependency for `next lint`. Never executed via glob CLI `-c` flag.
- **Vercel Applicability**: None (dev tooling only).
- **Final Disposition**: Deferred major `eslint-config-next` / Next.js 15+ breaking upgrade.

### 3. `next` — Framework Advisories (DoS, Image Optimizer, RSC, Proxy)
- **Advisory IDs**: GHSA-9g9p-9gw9-jx7f, GHSA-h25m-26qc-wcjf, GHSA-ggv3-7p47-pfv8, GHSA-3x4c-7xq6-9pq8, GHSA-q4gf-8mx6-v5v3, GHSA-8h8q-6873-q5fj, GHSA-3g8h-86w9-wvmq, GHSA-ffhc-5mcf-pf4q, GHSA-vfv6-92ff-j949, GHSA-gx5p-jg67-6x7h, GHSA-h64f-5h5j-jqjh, GHSA-c4j6-fc7j-m34r, GHSA-wfc6-r584-vfw7, GHSA-36qx-fr4f-26g5
- **Package**: `next` (9.3.4 - 14.2.35)
- **Installed Version**: 14.2.35
- **Dependency Path**: `next` (direct dependency)
- **Severity**: High
- **Affected Features**: Image Optimizer remote patterns, Server Components deserialization, i18n middleware proxy, WebSocket upgrades, CSP nonces.
- **Usage in WallMyDevice**: WallMyDevice operates entirely as a client-rendered canvas application. No Image Optimizer API, Server Components, or i18n proxy routes are utilized.
- **Vercel Applicability**: Application is deployed to Vercel as a static App Router page.
- **Final Disposition**: `ARCHITECTURE FIXES COMPLETE — NEXT.JS MAJOR SECURITY MIGRATION REQUIRED` (Framework upgrade to Next.js 15.5.21 or 16.2.11 deferred to distinct migration milestone to avoid breaking API changes).

### 4. `postcss` — Unescaped Style Tag XSS
- **Advisory ID**: GHSA-qx2v-qp2m-jg93
- **Package**: `postcss` (< 8.5.10)
- **Installed Version**: 8.4.38
- **Dependency Path**: `next` → `postcss`
- **Severity**: Moderate
- **Affected Feature**: XSS via unescaped `</style>` tag in CSS Stringify output.
- **Patched Version**: 8.5.10+
- **Usage in WallMyDevice**: Transitive sub-dependency of `next`. CSS stringify is evaluated only at build time.
- **Vercel Applicability**: None at runtime.
- **Final Disposition**: Bundled with `next`. Will resolve automatically upon Next.js framework migration.
